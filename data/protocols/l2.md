### L2 Specifics
- block.number on Optimism ≠ L1 block number (uses L2 block)
- Sequencer downtime: Chainlink feeds stale, auctions unfair
- CREATE2 addresses differ across chains if init code differs
- block.timestamp on Arbitrum may drift from L1
- Gas estimation differs: L1 data cost + L2 execution cost
- msg.sender in L1→L2 messages is aliased (address + offset)
